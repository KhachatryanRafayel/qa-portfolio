**ID:** BR-03\
**Status**: New\
**Severity:** Trivial
## Core Information
*   **Bug Type:** UI
*   **Build Version:** 1.0
*   **Environment:** Windows 11 home 24H2 / Python 3.13.1 / pygame  2.6.1
## Description
When entering a word or sequence longer than 40 characters, the text overflows outside the text field instead of remaining within the field boundaries.
## Preconditions
1. The game has started. (The user pressed any key.)
## Steps To Reproduce (STR)
1. Type a long word or sequence (>40 characters).
## Expected Result (ER)
*   The characters in the input field removes for the duration of the keypress.
## Actual Result (AR)
*   Character is deleted only once.
