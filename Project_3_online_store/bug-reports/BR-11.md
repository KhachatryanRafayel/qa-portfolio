ID: BR-11  
Related Test Case: PICKUP-33  

Summary: 
- Address input field does not validate maximum length, causing UI elements to shift when long text is entered
  
Environment:
- Windows 11 / Google Chrome
  
Expected Result:
- The search field accepts the full query without UI breaks. An error message (e.g., 'Street not found') should be displayed.
  
Actual Result:
- Users can enter an excessively long address. UI elements are displaced, breaking the layout.
  
Steps to reproduce:
1. Navigate to the "Homepage" -> <BASE_URL>
2. Click the "Specify address" at the top
3. Enter very long query (>500 digits) into the "Search on the map" field
   
Severity: Medium
