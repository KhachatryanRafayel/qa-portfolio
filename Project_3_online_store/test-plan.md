### 1. Project scope and main goals
- This test plan covers the **functional and UI testing** of main features of an e-commerce online store. The testing will focus on the front-end user experience and core user flows to ensure correct and stable functionality.
### 2. Requirements to be tested
The following features will be tested:
- Registration/Login
- Search and Filtering
- Favorites
- Cart
- Order Delivery Options
### 3. Requirements NOT to be tested
- Functionality "For the visually impaired"   :   Requires a specialized accessibility testing team.
- All goods categories   :   Out of testing scope due to limited time.
- Ordering and payment processes    :    Not tested due to lack of access to real or testing payment systems.
- Performance, security, or load testing   :   Requires specialized tools and expertise.
- Informational subsections of the website\`   :   Out of testing scope due to limited time.
	- About Ozon
	- The Ozon brand
	- Information for the media
	- Become a seller
	- Open the pick-up point
	- Help
### 4. Test strategy and approach
Considering that the testing team consists of only one person and given the limited time available, the following types of **manual testing** will be performed, focusing on the main functionality:
- Functional Testing
- UI Testing
### 5. Criteria
Entry Criteria:
- The website is accessible in the browser (Chrome)
- Test plan and checklists are ready
Exit Criteria:
- All planned test cases for main features (Cart, Search, Filters, Favorites, Language, Delivery Options) have been executed
- All bugs found are documented.
- Test documentation including checklists, test cases, bug reports and test summary report is completed.
